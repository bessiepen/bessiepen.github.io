CSS:包含块( containing block)

发表于 2016-03-04   |   分类于 技术笔记   |  

包含块是一个元素进行定位和尺寸计算的参照框，它是一个矩形。例如对元素p定位left:20px，那么这个left:20px是针对什么的左边20像素？那就是包含块的边界位置。

包含块的判定：

无position属性的元素（也就不涉及偏移量）：

元素的包含块是该元素上一级父元素块级框的内容边界（content edge），例子：

<html><body><div><p></p></div></body></html>

元素 包含块
html 初始包含块
body html
div body
p div

有position属性的元素：

  • static：不定位，left等无效，无包含块；
  • relative：包含块为该元素原本在的内容边界；
    • 例子：<p id=“relative”></p>，p元素的包含块是p元素初始内容框
  • fixed：
    • 设置偏移量时，包含块为当前视窗本身；
    • 未设置偏移量时：
      • 无已定位的祖先元素，会定位到最近父元素的左上角；
      • 有已定位的祖先元素
        • 祖先元素无偏移量，会定位到祖先元素的左上角。
        • 祖先元素有偏移量
          • 祖先元素为position：relative
            • 祖先元素是父元素时，偏移量不会继承下来，该元素还是定位到父元素的左上角固定不动；
            • 祖先元素是父元素以外的祖先元素时，偏移量会继承下来，但该元素以视窗为偏移。
          • 祖先元素为position：absolute/fixed
            • 不论父元素还是祖先元素有偏移量，会继承偏移量，但该元素以视窗为基准偏移。
  • position：
    • 设置偏移量时
      • 该元素附近有position值为static或无已定位的祖先元素，其包含块为初始包含块 <html>。
      • 该元素附近有position值为非static的祖先元素，包含块为该祖先元素：
        • 祖先元素是块级元素：包含块为祖先元素的padding edge形成；
        • 祖先元素是内联元素：包含块为祖先元素的内容边界形成。（direction:ltr/rtl，不同时判定不同，这点我还没搞清楚，以后遇到再补）。
    • 未设置偏移量时
      • 无已定位的祖先元素，本元素会定位到最近父元素的左上角；
      • 有已定位的祖先元素：
        • 祖先元素未设置偏移量，本元素会定位到祖先元素的左上角
        • 祖先元素已设置偏移量，本元素会继承偏移量，以祖先元素为偏移基准。

初始包含块

是根元素的盒子外边界组成，根元素是没有父元素的元素，即<html>。
