提问不是一件想当然的事

发表于 2016-07-24   |   分类于 阅读笔记   |  

（注，以下的黑客包括了已经是黑客和正在以黑客为目标努力的程序员）

花了一个星期通读了一遍《How To Ask Questions The Smart Way》（这真是一篇好长的文档，以下用《How To》代替），感觉自己的英文阅读能力稍微又进步了一点，果然用才是硬道理。才开始读的时候，是一段hacker的立场声明，hacker的那种看待世界的方式真是太吸引我了，太酷了，这也是我为什么想拐入编程学习的原因之一。看完这篇文档，收获大大，比起具体的方法论，更重要的是做事的方式问题。现整理如下。

note：《How To》是一篇专为学习计算机、编程（业余或专业）的人提供的一篇如何向黑客/开发者提问的指导型文档。它诞生于（我揣测的）太多人在提问这件事上漏洞百出，黑客们忍无可忍了，必需有责任亲自写一篇指导型文档，亲自教导真正想要解决问题的人如何正确地向自己提问。是的，我都写在这里了，你不照做，别怪我不回答你。

黑客文化是什么？

我们想要向黑客寻求帮助，首先要了解黑客们是如何处事的，“投其所好”，才可能得到他们的帮助。

之前看过（还未看完）一本好书《黑客与画家》，在序言中尤其澄清了下传统的真正的黑客是怎样的。

  • 在他们看来，如果要完成一个hack，就依然包含着高度的革新、独树一帜的风格、精湛的技艺。最能干的人会自豪地称自己为黑客。
  • “黑客”这个词不仅是第一流能力的象征，还包含着求解问题过程中产生的精神愉悦或享受。
  • 自由软件基金会创始人理查德·斯托尔曼说：“出于兴趣而解决某个问题，不管它有没有用，这就是黑客。”

这基本上决定了一个黑客愿不愿意解决你的问题取决于他对你的问题是否感兴趣，你的问题是否是有价值的.书中引用了另一本书《黑客：计算机革命的英雄》中提出的黑客伦理，这是我们理解黑客核心价值观的最佳论述：

黑客伦理：

  1. 使用计算机以及所有有助于了解这个世界本质的事物都不应受到任何限制。任何事情都应该亲手尝试。
  2. 信息应该全部免费。
  3. 不信任权威，提倡去中心化。
  4. 判断一名黑客的水平应该看他的技术能力，而不是看他的学历、年龄或地位等其他标准。
  5. 你可以用计算机创造美和艺术。
  6. 计算机使生活更美好。

以上概括成这几点：分享、开放、民主、计算机的自由使用、进步。

在《How To》中，除了计算机的自由使用这一点，其他都能反复找到痕迹。

（注：以下开始的引用内容，来自《How To》，但大多并非逐字翻译，而是我基于文档的总结概括）

■ 分享、开放

  • 不要发私人邮件，要公开写出来（如论坛），有助于让更多的人来帮助解决问题，也有助于给以后对这类问题有疑问的人提供参考。
  • 黑客文化倡导透明、开源，大家一起学习和互助。

■ 民主、进步

  • 是否已经具备能够吸引我们注意的技术性能力并不是必须的，必须的是你有一种最终可以导致这种能力的态度：警醒、思考、观察、愿意成为一个解决问题的参与者。
  • 好的问题可以帮助我们更好地思考，而且常常能够揭露那些我们也许还没有注意到或思考到的地方，所以如果你能问出一个好问题，我们非常感激。

但也还有另外一些重点观念，我在文档中反复发现。正因为黑客是这样一群对难题（他们的难题和我们以为的难题并不一定在同一水准上）如视珍宝、具有源源不断的探索精神、高难度思考需求的精英，因此他们要求高效率，要求能体现价值，他们不想把时间花在无意义的事情上，并且由于他们本身具有的黑客文化，一点都不介意他人对自己的看法，因此他们只关注问题，实事求是，不关注情绪或过于礼貌等对解决问题来说多余的累赘。

如：

  • 黑客更愿意理那些虽然不礼貌但是问题问得很有水平的人，也不愿意理那些非常礼貌却把问题说的不清不楚的人。
  • 不要问无语义无价值的问题，比如：有人能帮助我吗？简直多余（因为你问题问得好，自然有人回答，你问题问得错漏百出，黑客会直接无视）。
  • 过度自卑和自谦和唯唯诺诺对解决问题没有任何帮助，堂堂正正、客观事实地提出自己的问题，提供有价值的信息就行了。
  • 在论坛中不要滥用表情符号和富文本格式，让人感觉你关注方向有问题，你是来解决问题的还是来干嘛的？显得你很幼稚，像叽叽喳喳的小女孩。
  • 比起让别人觉得舒服，解决问题才是黑客们关心的事。所以不要介意他们的粗鲁。

■ 亲自动手

黑客非常崇尚实践，亲自动手。懒汉在他们眼中就是唯恐避之不及的时间陷阱：

  • 一定要花时间，解决一个问题不是那么简单的，至少要读很多资料，寻求帮助只是最后才该做的事。
  • 我们是这样一种人：对那些提问之前不愿意自己先思考，自己先试着做的人充满敌意。 这样的人简直是时间陷阱，他们只知道索取，不知道付出。

黑客是一群对自己有明确定义和明确价值观的人。这样的人知道自己在做什么，有很高和快速的抉择水平，但他们也是人，精力有限，百忙之中抽出一点时间来为他人解决问题，他们收到的邮件量是很大的，而没水准甚至情绪发泄的咨询邮件可能是压断骆驼背的最后一根稻草。因此，既然有问题的是我们，想要得到有用解决方法的是我们，就该摆正态度，努力为帮助自己解决问题的他们着想。没有谁在不付费的情况下有义务为你解决问题。

我们应该怎么向黑客提问

我们的问题想要被认可，必需符合一定的以上提到的黑客文化。也许每个人都有过这样的感受：有时有人向自己寻求帮助，却可以把自己气得够呛，也许每个人生气的原因不同，那是因为人人都有自己的处事原则，当这个提问者触犯了我们的处事原则，我们一点也不想帮助他。同理，黑客们更有一套属于自己的处事原则。

核心指导原则记住：

■ 凡是能在互联网上搜索到的就不要问人，这个原则贯穿询问前、询问中、询问后整个过程；

这个能在互联网上搜索到的程度是怎样的？是这样的：

  1. 搜索论坛中已存档的档案，那里也许有人问过同样的问题；
  2. 用google搜索网页；
  3. 查看手册，软件和产品通常都有自己的使用手册；
  4. 查看常见问题解答（FAQ）；
  5. 如果学的是编程，还可以阅读开源代码来寻求答案；
  6. 当你对回答你的人中的某个术语不懂时，不要问他，而是问互联网，实在搜索不到，再问他。

这样动手就行了吗？还不行。

■ 用尽一切办法准备你的问题

投入更多的努力，就可能得到更多有用的信息：

  1. 找对位置，在相关的专门网站寻找答案或提出问题；

    1. 不要在讨论windows的地方问mac的问题，其他同理；
    2. 不要在讨论基础问题的地方问高阶问题，反之亦然；
    3. 非技术性问题不要问开发论坛，要问用户论坛；

  2. 提出问题前先看看别人是怎样提问的，了解该论坛的提问风格；

  3. 注意问题格式；

    1. 标题很重要，以“对象-问题”的格式写，如“X.org 6.8.1 mouse cursor on Fooware MV1005 vid. chipset - is misshapen”，表明问题对象的版本号、使用环境，最后写问题，让人一目了然你的问题是什么；
    2. 不要犯拼写或语法的低级错误，除非你说明自己是非英语母语使用者，但是也要尽力，因为别人能一眼看出你到底是不会还是懒；
    3. 发送纯文本，不要富文本，不要附件；
    4. 表达务必达到即简洁又能把问题说清楚；
    5. 问题过长的，在开始写上摘要；
    6. 只发能保证体现问题的最短代码片，并指清第几行代码自己认为有问题；
    7. 非英语母语使用者，提供问题提供两个版本，母语和英语版；

  4. 把自己所做的努力写出来，查过哪些文档，做过哪些测试，这能让对方鉴定出你是否值得帮助。

■ 体贴入微，为他人节约时间

给我最大的启发就是这个概念，为他人着想，就是给自己方便，但是你远远不知道原来为别人着想是件多么细致入微的事。就像上文提到的“非英语母语使用者提供问题提供自己的母语和英文版两个版本”一样，是为了方便使用英语的人直接阅读。再者，如：

  • 对于非英语母语使用者，可以告诉对方，你可以用英语回答，我自己愿意翻译。
  • 不要发送word，excel等附件，给人增加下载下来再打开的麻烦步骤；
  • 如果程序有自我诊断，诊断过无问题的就排除掉，减轻黑客阅读负担；
  • 详述但不宽泛地叙述问题，为回答者节省理解和解决时间；
  • 如果你遇到一个有问题的代码，更好的方式是问别人它错在哪里，而不是让别人来解决它。这样就给别人节省了时间，他可以快速回答，你也能快速得到解题思路；
  • 在你的问题解决后，给所有帮助过你的人回复一个简短的信息，告知问题已解决；如果解决过程比较复杂，可概括重点，一目了然，同时给大家一个反馈以及互相学习的机会。

你想得越多，越为对方节省时间，减轻麻烦，就越能得到对方的及时帮助。

■ 认清自己，淡化情绪

最后，在做到了以上后，还有一个十分重要的事情需要认清：黑客没有义务来为我们解决问题，不要把别人的帮助看做理所当然，因此，摆正自己的位置，不要用情绪来思考，因为黑客关注的只是问题事实，如果他们以幽默的方式嘲笑我们，我们首先要反思的是自己是否问题问得太不认真，太理所当然？只知道索取，不知道付出？

因为，一个人可以无知，但不可以犯蠢。It’s OK to be ignorence, it’s not OK to stay stupid.

■ 最后，需要尤其谨慎的地方

  1. 不要滥发私人邮件；
  2. 不要一个渠道重复发邮件，不要一个邮件多个渠道发；
  3. 不要滥用大写，感觉你在嚎叫；
  4. 不要轻易注明“Urgent！”（紧急！）。

一个既没有提供有用信息，还叫嚣着急急急的人，真的非常令人反感。

去掉黑客这个标签，提问的一般性原则是什么？

虽然这篇文档更多的是为专业领域的提问提供指导，但是，我们可以从中提炼出一些普遍适用的提问原则。

今天去驾校考科一时，考前监考员口干舌燥地再三强调各种注意事项，尤其强调让我们现在一定要认真听，不要到时一上场一有任何问题就马上举手寻求帮助，监考员说，“一个考场50个人，一个早上200个人，你们动不动就提问，我们回答得过来吗？”“多动动脑筋，因为有些错误犯得实在太低级了（此句是我从一串串生动的例子概括总结的）”。监考员讲得很生气，看来是受够了这样的苦。

连考试也是要有这样一番考前指导的，提问的艺术无处不在，工作中更是时时发生。最怕遇到那种一点点小问题都要立刻咨询甚至喊你帮他解决的人，问之前，不能先自己想想吗？自己想不出来，不会网上搜索一下吗？网上都搜索不到，或搜到了还是不懂，这个时候才有资格问了。所以，人们大多数的生气来自于一个不付出的人却强迫你为他的懒惰投入时间和精力。

因此，总结几条提问的普适原则：

  1. 提问前，自己先思考，自己先动手，自己先搜索摸索；
  2. 提问时，处处为给你解答的人提供便利，节约他们理解和解答的时间；
  3. 解决后，给反馈，甚至是自己的收获，帮助你的人会由衷一笑；
  4. 更高级的，把自己得到的解决方案写下来，供自己复习或供将来有需要的人参考；
  5. 都做到了以上，最后一条感觉没必要了：不要理所当然，不要感情绑架，实事求是，关注问题本身。
